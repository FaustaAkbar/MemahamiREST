Get Started

Apa itu Backend ?

Backend adalah bagian "di belakang layar" dari sebuah aplikasi yang menangani logika, penyimpanan data, dan pemrosesan. Gampangnya backend itu bagian belakang dari aplikasi yang nggak keliatan, tapi dia yang ngatur semua supaya aplikasi bisa jalan dengan baik. Misalnya, kalau kita pakai aplikasi buat belanja, backend yang ngatur data produk dan pembayaran. Jadi, bisa dibilang backend itu kayak mesin di mobil: nggak kelihatan, tapi bikin semua berfungsi.

Pengelolaan Data

Backend itu bertugas menyimpan, mengolah, dan mengamankan data yang penting bagi aplikasi kita, seperti informasi pengguna, transaksi, atau produk. Backend memastikan data tersebut dapat diakses atau dimodifikasi saat diperlukan oleh aplikasi.

Logika Bisnis

Backend juga yang ngatur segala perhitungan atau proses yang terjadi di aplikasi. Misalnya, menghitung total belanja, memproses pembayaran, atau mengecek apakah stok produk masih tersedia.

Keamanan

Backend juga yang bertugas untuk menjaga keamanan data dan aplikasi. Dia memastikan hanya orang yang berhak yang bisa mengakses informasi tertentu, dengan memverifikasi identitas pengguna (autentikasi) dan memberi izin akses (otorisasi).

Apa itu REST API ?

Penjelasan Sederhana

Jadi, REST API itu ibarat cara komputer atau aplikasi ngobrol satu sama lain lewat internet. Mereka pakai protokol yang namanya HTTP, yang sering kita temui saat browsing di web. Bayangin aja kayak aplikasi di ponsel kamu yang butuh data dari server atau aplikasi lain supaya bisa jalan. Nah, REST API ini bikin itu semua jadi lebih gampang dan cepat, jadi aplikasi bisa saling tukar data dengan lancar. Biasanya sih, REST API sering dipakai di sistem yang pake arsitektur mikroservis, yang artinya tiap bagian aplikasi bisa nyambung dan kerja bareng tanpa ganggu satu sama lain.


Gimana Cara Kerjanya?

Mirip Pesan Makanan

Jadi gini, tiap kali kamu ngirim permintaan itu kayak kamu pesen makanan di restoran. Semua udah lengkap, tinggal diproses deh! Dapur (server) langsung ngerti apa yang harus disiapin

Komunikasi Tanpa Ribet

Kayak kamu pake menu di restoran, tinggal pilih apa yang diinginin, terus dapur langsung ngerti apa yang harus disiapin. Gampang kan?

Metode Standar yang Gampang

Biar makin jelas, misalnya kamu pesan (POST), cek pesanan (GET), ganti pesanan (PUT), atau batalin pesanan (DELETE). Simple banget kan?

Flow pada REST API

Aplikasi Kamu (Si Pelanggan)

Kamu yang ngajuin permintaan, kayak minta sesuatu yang dibutuhkan

API (Si Pelayan)

Pelayan ini yang nganterin permintaan ke tempat yang tepat, terus bawa balik jawaban

Database (Dapur)

Dapur yang nyimpen dan siapin data sesuai permintaan

Kenapa Harus Pakai REST API ?

  • 🌐 Bisa dipakai di semua bahasa pemrograman
  • 🚀 Cepat dan efisien
  • 🔒 Cara aman buat berbagi data

Metode HTTP di REST API

GET

Metode ini dipakai buat "ngambil" data dari suatu tempat. Jadi, kalau kamu cuma butuh lihat data, pakai GET aja. Ini kayak nanya ke toko, "Ada barang ini nggak?"

POST

Kalau kamu mau kirim data, misalnya buat isi form atau upload sesuatu, pakai POST. Ini kayak kasih order ke restoran, "Saya mau pesan ini, nih!"

PUT

Kalau data yang ada perlu diperbarui, pakai PUT. Jadi, kalau datanya udah ada, tinggal update aja. Misalnya, kalau kamu mau ganti alamat pengiriman, ya pakai PUT.

DELETE

Kalau kamu pengen ngapus data, tinggal pakai DELETE. Ini kayak bilang ke restoran, "Hapus pesanan saya!"
